## Intended behavior (full intent)

Cost and margin data is commercially sensitive — it exposes supplier landing prices, unit economics, and negotation leverage. The masking policy:

| Role | `cost_price` (raw ₹) | Derived margin % | Context |
|---|---|---|---|
| SUPERADMIN | visible | visible | Full P&L owner |
| ADMIN | visible | visible | Director-level |
| ACCOUNTANT | visible | visible | Finance/GST duty |
| CATALOG_MANAGER | visible on product-edit form **only** | visible on product-edit form only | Sets pricing; must see cost to set correct margin |
| AREA_MANAGER | **hidden** (show `"-"`) | **hidden** (show `"-"`) | DECISIONS.md: NO |
| STORE_MANAGER and below | **hidden** (show `"-"`) | **hidden** (show `"-"`) | Floor staff |

"Hidden" means: the API **strips** the field server-side so it is never in the JSON payload; the frontend renders `"-"` for any cell that receives `null`/absent cost or margin. A CATALOG_MANAGER calling `GET /catalog/products/{id}` for the edit form receives cost; calling `GET /inventory/stock` they do NOT (stock ledger is an operational view, not a pricing form).

No new DB collections, no schema changes. Pure-additive: a single service helper strips or nulls the field before the response leaves the router, and a single React component wraps every cost/margin cell.

## Backend (endpoints + services + which ENGINE calls)

No ENGINE calls. This feature has no dependency on E1/E2/E3/E4/E5/E6/PM/SC. It is a pure read-path filter.

**`backend/api/services/cost_mask.py` — complete contract:**

```python
COST_VISIBLE_ROLES = {"SUPERADMIN", "ADMIN", "ACCOUNTANT"}
CATALOG_FORM_ROLES = {"CATALOG_MANAGER"}
_COST_FIELDS = {"cost_price", "cost_value", "cost_at_sale"}
_MARGIN_FIELDS = {"margin_pct", "gross_margin", "net_margin", "cogs",
                  "gross_margin_pct", "net_margin_pct", "avg_margin_pct",
                  "total_cost", "cogs_estimated_lines"}

def can_see_cost(user: dict, context: str = "default") -> bool:
    roles = set(user.get("roles", []))
    if roles & COST_VISIBLE_ROLES:
        return True
    if context == "catalog_edit" and roles & CATALOG_FORM_ROLES:
        return True
    return False

def mask_cost(doc: dict, user: dict, context: str = "default") -> dict:
    if can_see_cost(user, context):
        return doc
    for f in _COST_FIELDS | _MARGIN_FIELDS:
        doc.pop(f, None)
    pricing = doc.get("pricing")
    if isinstance(pricing, dict):
        pricing.pop("cost_price", None)
    return doc

def mask_cost_list(docs: list, user: dict, context: str = "default") -> list:
    if can_see_cost(user, context):
        return docs
    return [mask_cost(d, user, context) for d in docs]
```

No new endpoints. All changes are in existing routers (catalog.py, inventory.py, analytics_v2.py, reports.py, finance.py). The function is a one-liner call in each router after the data is assembled, before the `return` statement.

**RBAC policy (`rbac_policy.py`):** no new entries needed; existing route gates are unchanged. The masking is orthogonal to access control (the route is still reachable; the field is just stripped from the response for unauthorized roles).
