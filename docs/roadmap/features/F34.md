## Intended behavior (full intent)

Every authenticated user who logs into IMS sees a **Target Ticker card** on the Hub/dashboard landing page. The card shows how close each store is to hitting its monthly revenue budget target (configured in the Budgets module). The display is privacy-stratified:

- Management (SUPERADMIN, ADMIN, AREA_MANAGER, STORE_MANAGER, ACCOUNTANT) see raw rupee amounts, numeric %, and a pace line (ahead/behind linear target for today's date).
- Store-floor roles (SALES_CASHIER, SALES_STAFF, CASHIER, OPTOMETRIST, WORKSHOP_STAFF, CATALOG_MANAGER) see only an unlabelled progress bar with "Company Goal — 72% reached" — no rupee amounts, no store breakdown.
- When no REVENUE budget exists for the current month, the card displays "No target set — configure in Budgets" and is visually greyed out. No fabricated number is ever shown.

When a store's MTD revenue crosses a milestone threshold (25%, 50%, 75%, 100% by default; owner-configurable via Settings), ORACLE's hourly tick detects the crossing and pushes an in-app bell notification to store-floor roles (SALES_CASHIER, SALES_STAFF, CASHIER) in that store — a brief celebratory message. Management tiers get no notification (option (b) from owner Q2, per DECISIONS §7 default-to-recommended). Milestones do not re-fire in the same calendar month. Bell only — no WhatsApp/SMS.

Milestone thresholds are global (option (a) from owner Q3), but stored as an E2-registered key `ticker.milestone_pcts` so a store-scoped override can be added later without a code change. Refresh interval is also an E2 key (`ticker.refresh_seconds`, default 60). Per-store milestone percentages are NOT built in v1; the E2 `scopes` on those keys already allows a future store-scope override.

The "No target set" fallback (option (a) from owner Q1) means a store without a REVENUE budget for the current month gets a greyed card rather than a fabricated number.
