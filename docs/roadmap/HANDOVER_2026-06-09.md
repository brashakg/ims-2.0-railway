## 3. The process that worked (keep doing this)

**Role:** you are BOTH the build and the orchestrator. A separate "tester" session validates + maintains the board in parallel (it self-merges direct sometimes — expect the board's DONE rows to fill in without you; rebase board PRs if they conflict).

**Per-feature loop:**
1. Read `docs/roadmap/{STATUS,PROTOCOL,DECISIONS,CORRECTIONS}.md` + the feature packet (`docs/roadmap/features/<id>.md`).
2. Build on a `feat/<id>-<slug>` branch — **launch agents via `Agent(isolation:"worktree", run_in_background:true)`** for parallel features; they commit to their own branch, do NOT push/PR/touch the board.
3. **Adversarially verify** with a `Workflow` (parallel `qa-code-auditor` lenses + a skeptical chair that re-checks each finding via `git show <branch>:<file>`). This catches the defects the agent's own green tests miss. **If the chair gets rate-limited (`synth:null`), do the adjudication yourself** from the raw lenses (read the code, confirm).
4. Fix confirmed P0/P1 **with a regression test that fails against the old code**; rebase onto current `main`; PR; **self-merge `--squash --delete-branch` only when required CI is green** (test 3.10/3.11, test-and-build 18.x/20.x, security, build). `e2e` + Vercel are **non-required cold-preview flakes** — UNSTABLE ≠ code failure.

## 4. Hard-won landmines (do not relearn these)

- **Branch protection does NOT block merges on red/pending required CI here.** Discipline is yours — verify required-checks-green before every merge. (PR #592 merged broken at a stale SHA; cost a cherry-pick repair via #596.)
- **CI-robustness in tests:** monkeypatch **every** repo/db accessor a handler uses **and seed every doc the atomic guard reads**. A fail-soft fallback that returns differently with no-Mongo (local) vs real-Mongo (CI) passes locally and **fails in CI** (bit E3w). `_orders_coll()` prefers `get_order_repository().collection` — patch that too.
- **tz trap (recurring):** `now_ist()` is tz-AWARE; pymongo `created_at`/`expires_at` are NAIVE → subtracting raises `TypeError`. Normalize with `_to_naive()` / `now_ist_naive()` / aware-UTC. And `entry_date` for accounting is a **calendar day** — do NOT route it through `ist_day_start_utc()` (it shifts to 18:30 UTC the prior day → wrong period-lock month + FY serial; this was #25's P1).
- **Never assert a secret/PIN's absence via a whole-JSON substring check** (see §2c flaky test) — assert on the specific field.
- **Standalone Mongo — zero multi-doc transactions.** Every balance/state change is a single guarded `find_one_and_update` whose **filter IS the guard** (mirror `vouchers.redeem_voucher_atomic` / `money_guard.debit`). For exactly-once money, pass an `idempotency_key` (money_guard honors the `money_ledger` marker → a duplicate returns `reason="duplicate"`, no second move).
- **§6 reviewer REJECTS in-feature reimplementation of shared engines.** Extend E1 money_guard / E2 get_policy / E4 ApprovalEngine / E6 reminder_rail / SC scorecard / campaign_segments — never fork them.
- **Numbering tangle:** `features/F17.md` is the **Petty-cash** packet; **maker-checker JE is roadmap #25** (`F25.md`). The JE shipped on a branch named `feat/F17-maker-checker-je` (#604) for continuity; petty-cash shipped as `feat/F17-petty-cash` (#607). Don't conflate them.
- **Merge-base artifact:** a feature branched before another merged will show the other's files as "deleted" in `git diff main...branch`. **Rebase onto current main** before merging — the files come back (F41 looked like it reverted F46; it didn't).
- **No emoji in Python** (Windows cp1252 crash). **Light theme only.** **Never print/commit secret VALUES** (keys only; `railway run` injects creds). **Never** `railway delete` at top level or `--admin` force-merge.
